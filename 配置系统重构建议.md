# Day Translation 配置系统重构建议

## 当前问题

目前存在两套配置系统，造成功能重复和管理混乱：

1. **config.py (TranslationConfig + user_config)**
   - 文件：`~/.day_translation/config.json`
   - 用途：翻译程序核心配置 + 一些用户设置（如API密钥）

2. **user_preferences.py (UserPreferences)**
   - 文件：`~/.day_translation/user_preferences.json`
   - 用途：用户操作偏好和界面设置

## 建议的配置系统分工

### 方案 A：明确分工（推荐）

#### config.py - 核心技术配置
**用途**：程序运行所需的技术参数
**文件**：`~/.day_translation/config.json`
**内容**：
- 语言设置（default_language, source_language）
- 目录结构（def_injected_dir, keyed_dir）
- 输出设置（output_csv, log_file）
- 调试模式（debug_mode）
- 过滤规则（filter rules）

#### user_preferences.py - 用户体验配置
**用途**：用户个人偏好和操作习惯
**文件**：`~/.day_translation/user_preferences.json`
**内容**：
- 提取偏好（merge_mode, structure_choice）
- 自动化设置（auto_mode, remember_paths）
- 路径记忆（remembered_paths, path_history）
- API密钥（API keys）← **移动到这里**
- 界面偏好（confirm_operations）

### 方案 B：完全整合

将所有配置合并到一个系统中，使用分层结构：

```json
{
  "version": "1.0.0",
  "core": {
    "default_language": "ChineseSimplified",
    "source_language": "English",
    "def_injected_dir": "DefInjected",
    "keyed_dir": "Keyed",
    "debug_mode": false
  },
  "user": {
    "extraction": {
      "merge_mode": "smart-merge",
      "structure_choice": "original",
      "auto_detect_en_keyed": true
    },
    "general": {
      "auto_mode": false,
      "remember_paths": true,
      "confirm_operations": true
    },
    "api": {
      "ALIYUN_ACCESS_KEY_ID": "",
      "ALIYUN_ACCESS_KEY_SECRET": ""
    },
    "paths": {
      "mod_dir": "/path/to/last/mod",
      "output_dir": "/path/to/output"
    }
  }
}
```

## 推荐实施步骤

我推荐**方案 A**，因为它：
1. 保持现有架构基本不变
2. 明确了各自的职责
3. 减少了重构风险
4. 便于维护

### 实施步骤：

1. **将 API 密钥管理移到 user_preferences.py**
2. **更新 interaction_manager.py 使用统一的偏好系统**
3. **明确文档说明两套配置的用途**
4. **可选：提供配置迁移工具**

## 具体改动建议

### 1. 扩展 user_preferences.py

```python
@dataclass
class ApiPreferences:
    """API配置偏好"""
    aliyun_access_key_id: str = ""
    aliyun_access_key_secret: str = ""
    save_api_keys: bool = True

@dataclass
class UserPreferences:
    """用户偏好的完整配置"""
    extraction: ExtractionPreferences
    import_prefs: ImportPreferences
    translation: TranslationPreferences
    api: ApiPreferences  # 新增
    general: GeneralPreferences
```

### 2. 更新 interaction_manager.py

```python
def show_welcome(self):
    """显示程序欢迎界面"""
    prefs = self.preferences_manager.get_preferences()
    has_api_key = bool(prefs.api.aliyun_access_key_id or os.getenv('ALIYUN_ACCESS_KEY_ID'))
    # ... 其他代码
```

### 3. 提供配置迁移

创建迁移工具，将现有的 `config.json` 中的用户相关设置迁移到 `user_preferences.json`。

## 优势

1. **职责明确**：技术配置 vs 用户偏好
2. **易于维护**：各自管理各自的领域
3. **向后兼容**：最小化现有代码改动
4. **扩展性好**：未来可以独立扩展各自的功能
